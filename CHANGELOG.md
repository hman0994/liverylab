## [Unreleased]

### Fixed
- `js/editor.js` — Canvas click-through bug: clicking a layer above the base layer in Select mode always selected the base layer instead of the topmost visible layer under the cursor.  Root cause was `_syncObjectInteractivity(target)` setting every object except the active one to `selectable: false`, preventing Fabric.js z-order hit-testing from reaching upper layers.  Now, in select mode, all user-editable objects stay `selectable: true` / `evented: true` so Fabric's built-in stacking-order click detection works naturally.  Protected layers (template, background, guides, spec-maps, hidden, locked) remain non-selectable.  Paint/shape/other tool modes keep the previous lock-everything behavior.
- `js/editor.js` — Properties panel retained stale selected-object values (opacity, colour, etc.) after switching tools because `setTool()` did not notify the workspace UI to re-read drawing defaults.  Now fires `onSelectionChanged(null)` at the end of `setTool()` so the Properties panel refreshes from `readWorkspaceState()`, which already returns correct defaults when the active tool is not `select`.
- `js/editor.js` — Layer opacity undo required two Ctrl+Z presses: the first committed the pending coalesced slider transaction, the second actually undid it.  Now `undo()` detects a pending coalesced transaction and discards (rolls back) instead of committing, so one Ctrl+Z returns to the pre-drag value.  Redo (Ctrl+Y) re-applies the discarded change correctly.
- `js/editor.js` — Duplicate layer undo required two Ctrl+Z presses because the async `target.clone()` callback could allow intermediate history snapshots between clone initiation and transaction commit.  Now `_suspendHistory` is held `true` across the entire async clone + insert path and only restored after the transaction commits.
- `js/editor.js` — Selection handles (corner resize + rotation) were nearly invisible because Fabric.js defaults use hollow/transparent corners in a faint colour.  Set prototype defaults for filled green circles (`cornerSize: 14`, `transparentCorners: false`, `cornerStyle: 'circle'`) with visible `cornerColor`/`borderColor` so handles are easy to grab on any background.
- `js/editor.js` — `hasControls` and `hasBorders` were not toggled by `_syncObjectInteractivity`, `_lockObjectsForPainting`, or `_unlockObjectsForPainting`, so controls could stay hidden after switching away from and back to the select tool.  Now explicitly set alongside `selectable`/`evented` and call `setCoords()` to recalculate hit-areas.
- `js/editor.js` — Objects jumped and rendered at the wrong position during drag/rotate because the viewport-base live overlay for `select-transform` kept a stale pre-composited image (including the target at its original position) beneath the transparent Fabric canvas, causing the target to appear doubled.  Disabled the live overlay for select-transforms; the viewport base is now suspended during object manipulation so Fabric renders all layers normally.
- `js/editor.js` — Delete layer threw "Cannot record a history operation without an active transaction" because `canvas.remove()` inside the `apply` callback fired Fabric's `object:removed` event synchronously, re-entering `_afterChange` → `_saveState` → `_flushPendingHistoryTransaction`, which committed and cleared the active transaction before `recordOperation` could run. Fixed by guarding the `apply()` call in `_runDocumentTransaction` with `_suspendHistory = true` to prevent re-entrant history capture.
- `js/editor.js` — Circle tool (`C` key) did not create any shape because `_createShapePreview('circle', …)` set a flag but never instantiated a `fabric.Ellipse`, falling through to `return null`. Now creates `new fabric.Ellipse({ …common, rx: 0, ry: 0 })` matching the `rx`/`ry` updates in `_updateShapePreview`.

### Added
- `js/app.js` — Escape key now deselects the active canvas selection. Skips if a modal is open (template picker, export) or the user is editing text inline (IText editing mode).
- `js/app.js` — Exposed `editor` on `window` so DevTools commands like `editor.getHistoryDebugState()` work from the console.
- **QA**: Created fresh v0.2.0 release QA checklist templates for Chrome and Edge under `docs/QA/Tests/v0.2.0-Release/`. Covers Sections 1–11 including architecture migration validation and render/export parity evidence table.
- **QA**: Updated `docs/QA/README.md` to point testers at the v0.2.0 release checklists.
